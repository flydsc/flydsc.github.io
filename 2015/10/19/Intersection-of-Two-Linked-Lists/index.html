Linked List Cycle

Linked List Cycle II

现在我们有了第一个题目的基础，也就是能够判断一个单链表中是否有环。接下来我们需要判断这个环的开头在哪里。这也就是需要求解的第二题：

Given a linked list, return the node where the cycle begins. If there is no cycle, return null.

Note: Do not modify the linked list.

Follow up:
Can you solve it without using extra space?

第二题的解题思路稍微复杂一些，我们假设这个链表是有环的，然后我们假设链表长度是L, 相遇的时候，相遇点距离环入口的距离是x, 链表起点距离环口的长度是a。我们假设在相遇的时候fast指针已经在环中，并且已经绕了n圈了。环的长度是r, 假设相遇的时候，快速指针走过的距离是2S, 慢速指针的距离是S，因此我们可以得到：

$$2S = S + nr$$
这个式子的理解就是，当两个指针相遇的时候，快速指针的距离相当于慢速指针的距离加上快速指针在环中走过的距离。

因此有：
$$S = nr$$

因为慢速指针走过的距离刚好等于链表起点到环口的位置的距离加上环口位置到相遇点的距离，所以可以得到：
$$a + x = nr$$
做一个变换
$$a + x = (n - 1)r + r$$
r是环的长度，整个链表是由环和非环的部分组成，所以可以得到
$$a + x = (n - 1)r + L - a$$
整理一下可以得到：
$$a = (n-1)r + (L – a – x)$$
a就是链表头到环口的距离，也就是我们要求的值。这个值等于两个部分加和，(n-1)r 相当于一个绕圈的加成数，
所以第二个部分可以变为：
$$L – a – x = a$$
现在就比较清楚了，L – a – x 就是两个指针相遇以后，再次到达环口的距离，这段距离刚好等于a，也就是链表开头到环口的距离。所以这个时候我们就想，如果确认了相遇点，那么我们这个时候在相遇点设一个指针，在开头设一个指针，让他们同步前进，那么他们必然会在环口相遇（因为式子中有(n - 1)r, 所以有可能是环内指针绕了几圈以后才相遇，但是和相遇点的参考距离没有变）
